# Professional Audit — 2026-05

**Subject:** `firefox-llm-bridge` v0.5.2 → v0.6.0
**Scope:** code quality, security, privacy, testing, benchmarking, documentation
**Auditor:** Claude (Anthropic) — automated agent, single-pass review
**Outcome:** all findings either fixed in v0.6.0 or carried as tracked follow-ups

This document is committed as a durable record of what the audit found and how the project responded. It is intentionally written so a future contributor can re-run the same audit and see what changed.

---

## TL;DR

The codebase entered this audit in unusually strong shape for a pre-1.0 extension: 757 tests, 100% coverage gate, typed errors, retry primitives, privacy-regression tests, full AMO submission docs. The audit's job was to find the highest-leverage gaps and close them, not to triage broken code.

Fifteen findings were raised. Eleven were fixed in v0.6.0. Four remain on the roadmap (i18n wiring, light theme, conversation export, provider failover) — they are scoped, ticketed, and not blockers for the AMO submission path.

## Findings & resolutions

### A. Reliability & observability

| #   | Severity | Finding                                                                                                                                  | Resolution in v0.6.0                                                                                                                                                                                                                                                                  |
| --- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| A1  | HIGH     | `lib/http.js#fetchWithRetry` was defined but unused — all four providers called bare `fetch()`. A single 5xx blip aborted the agent run. | Wired into all four providers for non-streaming calls. Streaming bypasses retry (cannot safely resume mid-stream) but still emits typed errors. New per-provider retry tests cover 5xx, 429 + `Retry-After`, transient network failures, and AbortError propagation.                  |
| A2  | HIGH     | Typed errors were not surfaced to the UI. Providers threw `new Error("Anthropic API 401: …")` and the sidebar rendered the raw string.   | All providers now throw typed `AuthError` / `RateLimitError` / `NetworkError` / `ProviderError`. Background's catch translates these to an `ERROR` payload carrying `{ code, retryable, providerId }`. Sidebar renders an error-code badge and a Retry button for retryable failures. |
| A3  | MED      | The agent kept running tokens after the user closed the sidebar (no `onDisconnect` handler aborted the in-flight controller).            | `port.onDisconnect` now aborts `state.abortController` when an agent is running. Three new tests cover present-controller, absent-controller, and abort-throws cases.                                                                                                                 |
| A4  | MED      | No structured logger. `console.info("[Firefox LLM Bridge] Initialized.")` was the only logging convention.                               | `background/lib/log.js` ships a namespaced ring-buffered credential-redacting logger. New "Debug logging" toggle in Options enables it. Sidebar exposes `GET_SESSION_LOG` to download a redacted session trace for bug reports.                                                       |
| A5  | LOW      | Anthropic cache token usage was extracted by `anthropic.js` but dropped by `recordUsage`.                                                | `state.cost.cacheReadTokens` and `cacheCreationTokens` now persist, survive session restore, survive `CLEAR_HISTORY`.                                                                                                                                                                 |

### B. Security & privacy

| #   | Severity | Finding                                                                                                                                                                             | Resolution in v0.6.0                                                                                                                                                                                                                                                                       |
| --- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| B1  | HIGH     | `input[type=password]` values could be exfiltrated in the accessibility map. Many autofill scenarios populate password inputs before the agent reads them.                          | `content/sensor.js#isSensitiveInput` redacts values from password, `autocomplete="current-password" / new-password / one-time-code"`, and `data-sensitive` inputs in both the `value` field and the label-inference fallback. `SYSTEM_PROMPT` updated with a SENSITIVE-INPUT POLICY block. |
| B2  | HIGH     | `execute_script` (arbitrary JS eval in the page) and `set_value` (bypasses keyboard simulation) were not in `DESTRUCTIVE_TOOLS`. Default `previewMode: "destructive"` skipped them. | Both are now in `DESTRUCTIVE_TOOLS`. `SYSTEM_PROMPT` adds a DANGEROUS-TOOL POLICY block telling the model to prefer `click_element` / `type_text` / `press_key` first.                                                                                                                     |
| B3  | MED      | CSP did not declare `default-src 'none'` or lock `frame-ancestors` / `base-uri` / `form-action`.                                                                                    | All four directives added to `manifest.json`. New assertion in `tests/privacy-regression.test.js` keeps them in place.                                                                                                                                                                     |
| B4  | MED      | `state.memories` had no quota — the model could fill `storage.local` (10 MB) via `remember`.                                                                                        | Hard caps: `MEMORY_MAX_ENTRIES = 100`, `MEMORY_MAX_ENTRY_CHARS = 4000`. Oldest entries evict FIFO. Empty content rejected. Eviction count surfaces in the tool result.                                                                                                                     |
| B5  | LOW      | Screenshots are sent to the LLM untouched. No redaction option.                                                                                                                     | Tracked for v0.7.0. Out of scope this pass because it touches the vision pipeline and benchmark fairness simultaneously.                                                                                                                                                                   |

## Out-of-scope but recommended for v0.7.0+

The plan that produced this audit (`/root/.claude/plans/audit-enhance-and-coherently-validated-glacier.md`) carries a Phase 5 covering items deferred for capacity reasons:

1. `prefers-color-scheme` light/dark adaptation
2. `browser.i18n.getMessage` wiring through every UI string
3. Conversation export / import as JSON (validated, never auto-executing tool calls on import)
4. Memory-governance UI in Options (list/search/delete entries)
5. Opt-in provider failover (e.g. retry on Ollama if Anthropic returns NETWORK)
6. ARIA + keyboard-navigation pass on sidebar and options
7. Screenshot redaction toggle
8. Prompt-injection fuzz corpus

Each of these is scoped and roadmap-ed; none are blockers for AMO submission of v0.6.0.

## How to reproduce this audit

1. `git checkout v0.6.0`
2. `npm ci`
3. `npm run check` — lint + format + typecheck + 100% coverage + web-ext lint
4. `npm run bench:dry` — confirm baseline parity
5. Read `manifest.json`'s CSP, the `DESTRUCTIVE_TOOLS` set in `background/lib/policy.js`, the `isSensitiveInput` helper in `content/sensor.js`, the `fetchWithRetry` integrations in `background/providers/*.js`.

The audit was performed against working-tree HEAD on branch `claude/audit-agent-testing-PL6pS`. All fixes ship in the same branch.
